Шпаргалка по Git для frontend-разработчиков

Инициализация нового проекта:

git init

Создает новый локальный git-репозиторий.

Клонирование существующего репозитория:

git clone

Получаем копию удалённого репозитория на ваш компьютер.

Просмотр статуса изменений:

git status

Показывает состояние файлов в рабочем каталоге и индексированной области.

Добавление файлов в staging area:

git add 
git add .

Добавляет файл или все файлы в область подготовки (staging area).

Создание коммита:

git commit -m "<сообщение>"

Фиксирует изменения с комментарием.

Просмотр истории коммитов:

git log

Показывает историю всех коммитов.

Переход на ветку или тег:

git checkout

Меняет активную ветвь или переходит на определенный тег/коммит.

Создание новой ветки:

git branch 
git checkout -b

Создает новую ветку либо сразу переключается на нее.

Объединение ветвей:

git merge

Интегрирует изменения одной ветви в текущую рабочую ветвь.

Отправка изменений на удалённый репозиторий:

git push origin

Выгружает локальные изменения на удаленный сервер.

Обновление изменений с удаленного сервера:

git pull

Синхронизирует вашу локальную версию с последней версией на удалённом сервере.

Отмена последнего коммита:

git reset HEAD~1 --hard

Удаляет последний коммит и сбрасывает рабочее дерево к предыдущему состоянию.

Изменение сообщения последнего коммита:

git commit --amend -m ""

Позволяет изменить комментарий к последнему коммиту.

Работа с конфликтами слияния:

git mergetool

Запускает графический интерфейс разрешения конфликтов.

Ежедневный сценарий работы:

  1. Начинаем рабочий день, обновляем проект:
    git pull
  2. Перед началом работы создаём новую ветку для конкретной фичи:
    git checkout -b feature/my-new-feature
  3. Работаем над задачей, добавляем изменения:
    git add .
  4. Комментируем изменения:
    git commit -m "Добавлена новая функциональность"
  5. Проверяем статус:
    git status
  6. Закидываем изменения на удалённый сервер:
    git push origin feature/my-new-feature
  7. Открываем Pull Request на GitHub/GitLab.
    После одобрения команды сливаем ветку обратно в main/master.
    git checkout main
    git merge feature/my-new-feature
  8. Чистим локальное окружение от старых веток:
    git branch -d feature/my-new-feature
